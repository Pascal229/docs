# 서드파티 애플리케이션

Logto의 서드파티 애플리케이션 통합을 통해 외부 애플리케이션에 대해 Logto를 [아이덴티티 제공자 (IdP)](https://auth.wiki/identity-provider)로 활용할 수 있습니다.

아이덴티티 제공자 (IdP)는 사용자 아이덴티티를 확인하고 로그인 자격 증명을 관리하는 서비스입니다. 사용자의 아이덴티티를 확인한 후, IdP는 인증 토큰 또는 어설션을 생성하여 사용자가 다시 로그인할 필요 없이 다양한 애플리케이션이나 서비스에 접근할 수 있도록 합니다.

[애플리케이션에 Logto 통합하기](/integrate-logto/integrate-logto-into-your-application) 가이드에서 생성한 애플리케이션과 달리, 서드파티 애플리케이션은 외부 개발자나 비즈니스 파트너가 개발한 독립적인 서비스입니다.

이 통합 접근 방식은 일반적인 비즈니스 시나리오에 적합합니다. 사용자가 Logto 계정을 사용하여 파트너 애플리케이션에 접근할 수 있도록 하거나, 엔터프라이즈 사용자가 Google Workspace로 Slack에 로그인하는 것처럼 "Logto로 로그인" 기능을 추가할 수 있는 오픈 플랫폼을 구축할 수 있습니다.

## Logto에서 서드파티 애플리케이션 생성하기 \{#create-an-third-party-application-in-logto}

1. <CloudLink to="/applications">콘솔 > 애플리케이션</CloudLink>으로 이동합니다.
2. 애플리케이션 유형으로 "서드파티 앱"을 선택하고 다음 통합 프로토콜 중 하나를 선택합니다:
   - OIDC / OAuth
   - SAML (곧 제공 예정)
3. 애플리케이션의 이름과 설명을 입력하고 "생성" 버튼을 클릭합니다. 새로운 서드파티 애플리케이션이 생성됩니다.

모든 생성된 서드파티 애플리케이션은 "서드파티 앱" 탭 아래의 애플리케이션 페이지에 목록화됩니다. 이 배열은 자체 애플리케이션과 구별할 수 있도록 도와주며, 모든 애플리케이션을 한 곳에서 쉽게 관리할 수 있게 합니다.
