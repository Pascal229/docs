# Naver 소셜 로그인을 설정하세요

Naver 커넥터는 애플리케이션이 Naver의 OAuth 2.0 인증 시스템을 사용할 수 있는 간결한 방법을 제공합니다.

<GuideTip />

## 개발자 사이트는 현재 한국어만 지원합니다 \{#developer-site-only-korean-support-now}

현재 `Naver Developers` 사이트는 한국어만 지원합니다. 번역기를 사용하는 것을 고려해 주세요.

## 프로덕션을 위한 설정 \{#for-the-production}

- 프로덕션을 위해서는 Naver 팀의 검토를 받아야 합니다.
  그렇지 않으면 등록된 사용자만 로그인할 수 있습니다.
  - `맴버관리(Member Manage)` 메뉴에서 테스터를 추가할 수 있습니다.
- 검토를 받으려면 애플리케이션 프로젝트 설정에서 `API 설정(API Setting)`의 `애플리케이션 개발 상태(Application Devlopment Status)`를 확인하세요.

## Naver Developers에서 프로젝트 설정하기 \{#set-up-a-project-in-the-naver-developers}

- [Naver Developers](https://developers.naver.com/main/)를 방문하여 Naver 계정으로 로그인하세요.
- 메뉴에서 **Application -> 어플리케이션 등록**을 클릭하여 새 프로젝트를 만드세요.
- 아래 지침에 따라 애플리케이션을 만드세요.

### 어플리케이션 이름 \{#application-name-어플리케이션-이름}

- `어플리케이션 이름`에 애플리케이션 이름을 입력하세요 (이 이름은 사용자가 로그인할 때 표시됩니다).

### 사용 API \{#api-usage-사용-api}

- `사용 API(API Usage)`로 `네이버 로그인(Naver Login)`을 선택하세요.
- `권한(Role)`에서 `이메일 주소(Email Address), 별명(Nickname), 프로필 사진(Profile Image)`을 `필수(Neccessary)`로 체크하세요 (이 옵션들을 선택적으로 `추가(Add)`로 체크할 수 있지만, 사용자로부터 정보를 얻을 수 없습니다).

### 로그인 오픈 API 서비스 환경 \{#sign-in-open-api-service-environment-로그인-오픈-api-서비스-환경}

- `로그인 오픈 API 서비스 환경(Sign in Open API Service Environment)`에 `PC웹(PC Web)`과 `모바일웹(Mobile Web)` 두 환경을 추가하세요.

#### PC 웹 \{#pc-web-pc-웹}

- `서비스 URL(Service URL)`에 `http(s)://YOUR_URL`을 입력하세요 (예: https://logto.io)
- `네이버 로그인(Naver Login) Callback URL`에 `http(s)://YOUR_URL/callback/${connector_id}`를 입력하세요 (예: https://logto.io/callback/${connector_id})

#### 모바일 웹 \{#mobile-web-mobile-웹}

- `서비스 URL(Service URL)`에 `http(s)://YOUR_URL`을 입력하세요 (예: https://logto.io)
- `네이버 로그인(Naver Login) Callback URL`에 `http(s)://YOUR_URL/callback/${connector_id}`를 입력하세요 (예: https://logto.io/callback/${connector_id})

:::caution

`connector_id`는 Logto Admin Console 커넥터 세부 정보 페이지의 상단 바에서 찾을 수 있습니다.

:::

## Logto 구성하기 \{#configure-logto}

### 구성 유형 \{#config-types}

| 이름         | 유형   |
| ------------ | ------ |
| clientId     | string |
| clientSecret | string |

#### clientId \{#clientid}

`clientId`는 프로젝트의 `Client ID`입니다.
(Naver developers의 프로젝트 `애플리케이션 정보(Application Info)`에서 찾을 수 있습니다.)

#### clientSecret \{#clientseceret}

`clientSecret`는 프로젝트의 `Client Secret`입니다.
(Naver developers의 프로젝트 `애플리케이션 정보(Application Info)`에서 찾을 수 있습니다.)
